# IMP Build Workflow

This document describes the complete build pipeline for the Inference Machine Pipeline (IMP), converting Brief specifications to executable binaries for the AMD/Xilinx Kria KV260.

## Overview

```
Brief Source (.ebv)    →    Rust / SystemVerilog    →    Binary
                              │                      │
                        ┌──────┴──────┐       ┌─────┴─────┐
                        ▼             ▼       ▼           ▼
                   ARM Target    FPGA Target   .elf       .bit/.bitstream
```

## Prerequisites

- brief-compiler (built from `/brief-compiler`)
- ARM toolchain: `arm-none-eabi-gcc` or `cargo build --target arm-unknown-linux-gnueabihf`
- Xilinx Vivado (for FPGA synthesis)

## Command Reference

### `brief-compiler arm <file.ebv> [--out <dir>]`

Compiles Embedded Brief to ARM-compatible Rust (bare-metal, no_std).

**Options:**
- `--out <dir>` - Output directory (default: `<stem>.rs`)

**Example:**
```bash
brief-compiler arm kernel.ebv --out arm/
```

### `brief-compiler verilog <file.ebv> --hw <hardware.toml> [--out <dir>]`

Compiles Embedded Brief to SystemVerilog for FPGA synthesis.

**Options:**
- `--hw <hardware.toml>` - **Required** hardware configuration
- `--out <dir>` - Output directory

**Example:**
```bash
brief-compiler verilog neuralcore.ebv --hw hardware.toml --out fpga/
```

### `brief-compiler rbv <file.rbv> [--out <dir>]`

Compiles Rendered Brief to WebAssembly (browser).

## Build Steps

### 1. Software (ARM Kernel)

```bash
# Compile Brief to ARM Rust
./brief-compiler arm kernel.ebv --out arm/

# Add bare-metal runtime (minimal startup + hardware glue)
# This requires additional code - not generated by the compiler

# Compile to ELF
arm-none-eabi-gcc -o kernel.elf \
  -T linker_script.ld \
  -nostdlib \
  -ffreestanding \
  arm/kernel.rs \
  start.o
```

**Output:** `kernel.elf`

### 2. Hardware (FPGA)

```bash
# Compile Brief to SystemVerilog
./brief-compiler verilog neuralcore.ebv --hw hardware.toml --out fpga/

# Synthesize with Vivado
cd fpga
vivado -mode batch -source synth.tcl

# Generate bitstream
vivado -mode batch -source bitstream.tcl
```

**Output:** `neuralcore.bit` or `neuralcore.bitstream`

### 3. Boot Image

```bash
# Create BOOT.BIN for KV260
# Based on Zynq UltraScale+ boot flow

# 1. FSBL (First Stage Bootloader)
# 2. ATF (Arm Trusted Firmware)
# 3. U-Boot
# 4. FPGA bitstream
# 5. kernel.elf

mkimage -A arm -T kernel -C none -a 0x00000000 -e 0x00000000 \
  -n "IMP Kernel" -d kernel.elf uImage
```

## Memory Map

| Address | Region | Type | Purpose |
|---------|--------|------|---------|
| 0x0000_0000 | DDR4 Base | DDR4 | Kernel + Runtime |
| 0x4000_0000 | Control | Flip-Flop | AXI4-Lite registers |
| 0x4000_4000 | KV Cache | UltraRAM | Neural network state |
| 0x4008_0000 | Tensors | BRAM | Input/output buffers |
| 0x1000_0000 | Weights | DDR4 | Pre-loaded model weights |

## Troubleshooting

### "Proof errors (continuing anyway)"

The ARM backend skips proof verification by default. This is expected for hardware interface code.

### "Hardware config required"

For SystemVerilog generation, you must provide `--hw hardware.toml`.

### ELF not executing

- Ensure `kernel.elf` is linked at correct address (0x0000_0000 for KV260)
- Verify U-Boot boot command loads correct address
